Toast App Configuration Schema

This document outlines the configuration schema for Toast App, detailing all available options and their formats. The configuration file is stored as JSON and can be edited directly or through the Settings UI.

Configuration Overview

Toast App's configuration is stored as a JSON file, which defines all user preferences, buttons, pages, and application behavior. The configuration can be edited directly using a text editor or through the Settings UI in the application.

The configuration follows a hierarchical structure:

  • Top-level properties define global settings like the global hotkey.
  • The pages array defines groups of buttons, each with its own settings.
  • Each page contains a buttons array with individual button definitions.
  • Buttons define their actions, which vary based on the action type.
  • The appearance object controls UI settings.
  • The advanced object contains system behavior settings.

File Location

The configuration file is stored in a platform-specific location:

macOS

~/Library/Application Support/toast-app/config.json

Windows

%APPDATA%\toast-app\config.json

The file is created automatically with default values when Toast App is first launched. If the file becomes corrupted or is deleted, Toast App will generate a new configuration file with default values.

Top-Level Properties

globalHotkey

String Required

The global keyboard shortcut that triggers the Toast popup. Must be a valid Electron accelerator string.

Default Value:

"Alt+Space"

Valid examples:

  • "Alt+Space"
  • "Ctrl+Shift+T"
  • "CommandOrControl+P"

pages

Array Required

An array of page objects, each defining a set of buttons.

Default Value:

[]

See Pages Schema for details on page object structure.

appearance

Object Optional

Object containing appearance settings for the Toast UI.

Default Value:

{ "theme": "system", "position": "center", "size": "medium" }

See Appearance Schema for details on appearance object structure.

advanced

Object Optional

Object containing advanced settings for Toast App behavior.

Default Value:

{ "launchAtLogin": true, "hideAfterAction": true, "hideOnBlur": true, "hideOnEscape": true }

See Advanced Settings Schema for details on advanced object structure.

version

String Optional

The version of the configuration schema. This is used for automatic config migration when the schema changes between app versions.

Default Value:

"1.0.0"

Pages Schema

Each page object in the pages array defines a set of buttons that can be accessed through tab navigation in the Toast popup.

name

String Required

The display name of the page. This is shown in the page tabs.

shortcut

String Required

The keyboard shortcut to access this page when the Toast popup is open. Typically a single number (1-9).

buttons

Array Required

An array of button objects for this page.

Default Value:

[]

See Buttons Schema for details on button object structure.

icon

String Optional

An icon for the page tab. Can be an emoji or a named icon.

Buttons Schema

Each button object in a page's buttons array defines an action that can be triggered from the Toast popup.

name

String Required

The display name of the button. This is shown on the button in the Toast popup.

shortcut

String Required

The keyboard shortcut to trigger this button when the Toast popup is open. Typically a single letter.

icon

String Optional

An icon for the button. Can be an emoji or a named icon.

action

String Required

The type of action to perform. Must be one of: "exec", "open", "shortcut", "script".

Additional properties depend on the action type. See Action Schemas for details.

Action Schemas

Different action types require different additional properties in the button object.

Exec Action

The "exec" action executes a shell command.

command

String Required

The shell command to execute.

workingDirectory

String Optional

The directory in which to execute the command.

Default Value:

User's home directory

runInTerminal

Boolean Optional

Whether to run the command in a terminal window.

Default Value:

false

Open Action

The "open" action opens a URL or file.

url

String Required

The URL or file path to open.

application

String Optional

The application to use for opening the URL or file. If not specified, the system default application will be used.

Shortcut Action

The "shortcut" action simulates pressing a keyboard shortcut.

keys

String Required

The keyboard shortcut to simulate (e.g., "Ctrl+C", "Alt+Tab", "Cmd+Space").

Script Action

The "script" action executes a custom script.

scriptType

String Required

The type of script. Must be one of: "javascript", "applescript" (macOS only), "powershell" (Windows only), "bash" (macOS/Linux only).

script

String Required

The script content to execute.

Appearance Schema

The appearance object defines visual settings for the Toast UI.

theme

String Optional

The visual theme to use. Must be one of: "system" (follow OS setting), "light", "dark".

Default Value:

"system"

position

String Optional

The position of the Toast popup on the screen. Must be one of: "center", "top", "bottom", "cursor".

Default Value:

"center"

size

String Optional

The size of the Toast popup. Must be one of: "small", "medium", "large".

Default Value:

"medium"

opacity

Number Optional

The opacity of the Toast popup. Must be a number between 0.1 and 1.0.

Default Value:

1.0

buttonLayout

String Optional

The layout of buttons in the Toast popup. Must be one of: "grid", "list".

Default Value:

"grid"

Advanced Settings Schema

The advanced object defines system behavior settings for Toast App.

launchAtLogin

Boolean Optional

Whether to start Toast App automatically when the user logs in.

Default Value:

true

hideAfterAction

Boolean Optional

Whether to hide the Toast popup after executing an action.
